Como funciona

As conexões IPsec incluem os seguintes passos:

Troca de chaves: as chaves são necessárias para criptografia; uma chave é composta por uma sequência de caracteres aleatórios que podem ser usados para "bloquear" (criptografar) e "desbloquear" (descriptografar) mensagens. O IPsec configura chaves com uma troca de chaves entre os dispositivos conectados, para que cada dispositivo possa descriptografar as mensagens do outro dispositivo.

Cabeçalhos e trailers de pacotes: todos os dados que são enviados por uma rede são divididos em pedaços menores chamados pacotes. Os pacotes contém um conteúdo ou os dados reais que estão sendo enviados e cabeçalhos, ou informações sobre esses dados, para que os computadores que recebem os pacotes saibam o que fazer com eles. O IPsec, adiciona vários cabeçalhos aos pacotes de dados que contém informações de autenticação e criptografia. O IPsec também adiciona trailers, que seguem após o conteúdo de cada pacote ao invés de seguir antes.

Autenticação: O IPsec fornece autenticação para cada pacote, como um selo de autenticidade em um item colecionável. Isso garante que os pacotes sejam de uma fonte confiável e não de um invasor.

Criptografia: O IPsec criptografa os conteúdos dentro de cada pacote e o cabeçalho IP de cada pacote (a menos que o modo de transporte seja usado em vez do modo de túnel). Isso mantém os dados enviados por IPsec seguros e privados.

Transmissão: os pacotes de IPsec criptografados viajam por uma ou mais redes até seu destino usando um protocolo de transporte. Nesse estágio, o tráfego IPsec difere do tráfego IP regular, pois na maioria das vezes usa UDP como protocolo de transporte, em vez de TCP. O TCP, Protocolo de Controle de Transmissão, configura conexões dedicadas entre dispositivos e garante que todos os pacotes cheguem. O UDP, Protocolo de Datagrama do Usuário, não configura essas conexões dedicadas. O IPsec usa o UDP porque isso permite que os pacotes de IPsec passem pelos firewalls.

Descriptografia: na outra extremidade da comunicação, os pacotes são descriptografados e os aplicativos (por exemplo, um navegador) agora podem usar os dados entregues.

Protocolos usados no IPsec:

Cabeçalho de Autenticação (AH): fornece proteção à integridade assim como autenticação da origem dos dados, de controle de acesso e, opcionalmente, proteção a ataque replay. O modo de transporte, usado para proteger comunicações par-a-par (peer-to-peer), protege dados de camadas superiores assim como as partes do cabeçalho IP que não variam de maneira imprevisível durante a entrega do pacote. O modo de túnel pode ser usado para proteger tráfegos gateway-to-gateway ou host-to-gateway, assim como pode ser usado para tráfego host-to-host. Esse tipo de AH protege a parte interna (original) do cabeçalho e dados das camadas superiores assim como as partes do cabeçalho externo (túnel) que não variam de forma imprevisível durante a entrega do pacote. Como partes do cabeçalho IP não estão incluídas nos cálculos do AH, seu processamento é mais complexo que o processamento ESP. AH também não funciona na presença de NAT (Network Address Translation). Para IPsec-v3, AH não é necessário, porém para IPsec-v2 é.

No modo de transporte, AH é inserido depois do cabeçalho IP e antes de um protocolo de camada superior, ex.: TCP, UDP, ICMP,... ou antes de qualquer outro cabeçalho IPsec que já foi inserido. No contexto de IPv4 é recomendado colocar AH antes do cabeçalho IP (e todas as opções que contém), mas antes do protocolo de camada superior.

No contexto IPv6, AH é visto como um payload fim-a-fim, então deveria aparecer após os cabeçalhos salto-por-salto, de roteamento e a extensão de fragmentação. Os cabeçalhos de extensão de opções de destino podem aparecer antes ou depois do cabeçalho AH, dependendo da semântica desejada.

O modo túnel AH pode ser empregado por hospedeiros ou gateways de segurança (implementações "bump-in-the-stack" ou "bump-in-the-wire"). Quando AH é implementado em um gateway de segurança (para proteger o trânsito de tráfego), o modo túnel deve ser usado. No modo túnel, o cabeçalho IP “interno” carrega os endereços finais de origem e destino, enquanto um cabeçalho IP “externo” pode conter endereços IP distintos, ex.: endereços de gateways de segurança.

Protocolo de Encapsulamento de Segurança (ESP): fornece confidencialidade, integridade, autenticidade, controle de acesso e, opcionalmente, proteção contra ataque de replay e de análise de tráfego. O modo de transporte ESP protege dados das camadas superiores, mas não o cabeçalho IP. O modo de túnel ESP protege os dados de camadas superiores e o cabeçalho interno, mas não o externo.

Associação de Segurança (SA): se refere a vários protocolos usados para negociar chaves e algoritmos de criptografia. Um dos protocolos SA mais comuns é o de Troca de chaves da internet (IKE).
Por fim, enquanto o protocolo de internet (IP) não faz parte do pacote IPsec, o IPsec é executado diretamente sobre o IP.
